- Testing

--------------------------------------------------
DEFINITION OF DONE
--------------------------------------------------

A user story is considered DONE when:
- Code is completed and reviewed
- Unit testing is done
- No critical bugs
- Feature deployed to test environment
- Product Owner approval received

--------------------------------------------------
SPRINT ACTIVITIES
--------------------------------------------------

- Daily Scrum (15 minutes)
- Sprint Review (Demo)
- Sprint Retrospective (Improvement discussion)

--------------------------------------------------
INTERVIEW KEY POINT
--------------------------------------------------

Sprint planning involves selecting high-priority backlog items based on team velocity,
defining a sprint goal, breaking stories into tasks, and committing as a team.
